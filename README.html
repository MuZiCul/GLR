GLR

Green Leaf Ratio Analyzer / 绿叶比率分析器

此Java应用程序通过分析图片中HLS（色相、亮度、饱和度）颜色空间内指定绿色范围的像素百分比来计算"绿视率"。

This Java application analyzes images to calculate the "green visual rate" - the percentage of pixels in an image that fall within a specified green color range in HLS (Hue, Lightness, Saturation) color space.

Technical Approach / 技术路线

  1. 图像输入：从指定路径读取图像文件
    Image Input: Reads an image file from the specified path
  2. 预处理：可选地使用缩放因子调整图像大小以提高性能
    Preprocessing: Optionally resizes the image using a scaling factor for performance
  3. 色彩转换：使用自定义实现将每个像素从RGB转换为HLS色彩空间
    Color Conversion: Converts each pixel from RGB to HLS color space using a custom implementation
  4. 绿色检测：基于HLS阈值识别绿色像素
    Green Detection: Identifies green pixels based on HLS thresholds:
    • 色调(H)：52-152（HLS中的绿色范围） Hue (H): 52-152 (green range in HLS)
    • 饱和度(S)：0.13-1.0 Saturation (S): 0.13-1.0
    • 亮度(L)：0.1-0.9 Lightness (L): 0.1-0.9
  5. 计算：计算绿色像素与总像素的比率并以百分比表示
    Calculation: Computes the ratio of green pixels to total pixels as a percentage

Key Features / 主要特性

  • 自定义RGB到HLS转换算法（GitHub上最早的Java实现之一）
  • Custom RGB to HLS conversion algorithm (one of the first Java implementations on GitHub)
  • 可配置的缩放因子，用于高效处理大图像
  • Configurable scaling factor for processing large images efficiently
  • 简单的命令行界面
  • Simple command-line interface
  • 自包含，依赖极少（仅Java AWT）
  • Self-contained with minimal dependencies (Java AWT only)

Color Space Explanation / 色彩空间说明

此应用程序使用HLS（色相、亮度、饱和度）色彩空间而不是RGB，因为：

  • 色相更能代表色彩感知
  • 将色彩信息（色相）与亮度（亮度）和纯度（饱和度）分离
  • 定义色彩范围更直观（例如，"绿色"是特定的色相范围）

The application uses HLS (Hue, Lightness, Saturation) color space rather than RGB because:

  • Hue better represents color perception
  • Separation of color information (Hue) from brightness (Lightness) and purity (Saturation)
  • More intuitive for defining color ranges (e.g., "green" is a specific hue range)

Custom RGB to HLS Implementation / 自定义RGB到HLS实现

RGB2HLS.java文件包含RGB到HLS转换算法的自定义实现。根据项目注释，这是GitHub上最早的此转换的Java实现之一。

The RGB2HLS.java file contains a custom implementation of the RGB to HLS conversion algorithm. According to the project comments, this was one of the first Java implementations of this conversion available on GitHub.

Limitations and Future Improvements / 局限性和未来改进

  1. 硬编码参数：图像路径和缩放因子当前是硬编码的
  2. Hardcoded Parameters: Image path and scaling factor are currently hardcoded
  3. 无命令行界面：接受参数以提高灵活性将是有益的
  4. No Command-Line Interface: Would benefit from accepting arguments for flexibility
  5. 错误处理有限：图像加载的基本错误处理
  6. Limited Error Handling: Basic error handling for image loading
  7. 性能：可以针对非常大的图像进行优化
  8. Performance: Could be optimized for very large images
